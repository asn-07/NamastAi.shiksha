1. Identifiers in Java

A name in Java program is called identifier. It may be class name, method name, variable name or label name.

üìù Rules for Java Identifiers:

  1. Allowed characters: a-z, A-Z, 0-9, underscore (_), dollar ($)
  2. Cannot start with digit: ABC123 (valid) but 123ABC (invalid)
  3. Case sensitive: number, Number, and NUMBER are different
  4. No length limit: But recommended to keep under 15 characters
  5. Cannot use reserved words: int if = 10; (invalid)
  6. Can use predefined class names: int String = 10; (valid but not recommended)
public class Test {
  public static void main(String[] args) {
    int total_number = 100; // ‚úÖ valid identifier
    int Total# = 50; // ‚ùå invalid - contains #
    int 123abc = 25; // ‚ùå invalid - starts with digit
  }
}
üí° Note: While int String = 10; is syntactically valid, it's considered bad practice as it reduces code readability.

2. Reserved Words

Reserved words are special identifiers that have predefined meaning in Java.

Category Keywords Count
Data Types byte, short, int, long, float, double, char, boolean 8
Flow Control if, else, switch, case, default, for, do, while, break, continue, return 11
Modifiers public, private, protected, static, final, abstract, synchronized, native, strictfp, transient, volatile 11
Exception Handling try, catch, finally, throw, throws, assert 6
Class Related class, package, import, extends, implements, interface 6
Object Related new, instanceof, super, this 4

üóùÔ∏è Key Points about Reserved Words:

  • All reserved words contain only lowercase alphabet symbols
  • goto and const are reserved but unused
  • strictfp added in 1.2v, assert in 1.4v, enum in 1.5v
  • Reserved literals: true, false, null
  • void indicates no return type (mandatory in Java)

4. Literals

Any constant value which can be assigned to a variable is called a literal.

üî¢ Integral Literals:

  • Decimal: 10, 20, 30 (digits 0-9)
  • Octal: 010, 0777 (prefix with 0, digits 0-7)
  • Hexadecimal: 0x10, 0xFACE (prefix with 0x, digits 0-9, A-F)
  • Binary (Java 1.7+): 0b101, 0B111 (prefix with 0b)

üìù Character & String Literals:

// Character literals
char ch1 = 'a'; // ‚úÖ valid
char ch2 = '\n'; // ‚úÖ escape character
char ch3 = '\u0061'; // ‚úÖ Unicode for 'a'
char ch4 = 97; // ‚úÖ decimal Unicode value

// String literals
String s1 = "Hello"; // ‚úÖ valid
String s2 = "Line1\nLine2"; // ‚úÖ with escape sequence
üí° Java 1.7 Enhancement: You can use underscore in numeric literals for better readability: int million = 1_000_000; or double pi = 3.14_15_92;

5. Arrays

An array is an indexed collection of fixed number of homogeneous data elements.

‚úÖ Advantages:

  • Represent multiple values with single name
  • Improves code readability
  • Better performance for fixed-size collections

‚ùå Disadvantages:

  • Fixed size (cannot grow or shrink)
  • Homogeneous elements only
  • Memory wastage if size overestimated
// Array Declaration
int[] a1; // ‚úÖ recommended style
int a2[]; // ‚úÖ valid but less readable

// Array Creation
int[] arr = new int[5]; // creates array of size 5

// Array Initialization
int[] numbers = {10, 20, 30, 40, 50};

// Multi-dimensional Array
int[][] matrix = {{1, 2}, {3, 4}};
‚ö†Ô∏è Important Array Facts:
  • Array size must be non-negative integer
  • Arrays with size 0 are legal: int[] a = new int[0];
  • Negative array size causes NegativeArraySizeException
  • length is a variable, not a method: arr.length
  • Anonymous arrays: new int[]{10, 20, 30}

6. Variables in Java

Based on behavior and position of declaration, variables are categorized into three types.

Type Declaration Storage Default Value Scope Thread Safety
Instance Variables Inside class, outside methods Heap Yes (JVM provides) Object lifetime ‚ùå Not Thread safe
Static Variables With static modifier Method Area Yes (JVM provides) Class lifetime ‚ùå Not Thread safe
Local Variables Inside methods/blocks Stack No (must initialize) Block lifetime ‚úÖ Thread safe

üîë Key Differences:

  • Instance Variables: One copy per object, created with new
  • Static Variables: One copy per class, created at class loading
  • Local Variables: Created when block executes, destroyed when block ends
  • Only final modifier can be used with local variables
  • Local variables must be initialized before use

7. Var-arg Methods (Java 1.5+)

Methods that can accept variable number of arguments of the same type.

// Var-arg method declaration
public static void methodOne(int... x) {
  for (int i : x) {
    System.out.println(i);
  }
}

// Calling var-arg method
methodOne(); // ‚úÖ no arguments
methodOne(10); // ‚úÖ one argument
methodOne(10, 20, 30); // ‚úÖ multiple arguments

üìã Rules for Var-arg Methods:

  1. Only one var-arg parameter per method
  2. Var-arg parameter must be the last parameter
  3. Valid syntax: int... x, int ...x, int...x
  4. Invalid syntax: int x...
  5. Internally implemented using single-dimensional array
  6. Var-arg methods get least priority in overloading
‚ö†Ô∏è Cannot overload with array parameter:
method(int... i) and method(int[] i) will cause compile-time error as they are essentially the same.

8. Main Method

The entry point for Java application execution.

// Standard main method signature
public static void main(String[] args) {
  // Program execution starts here
}

// Valid variations
static public void main(String[] args) { } // ‚úÖ order doesn't matter
public static void main(String... args) { } // ‚úÖ var-arg parameter
public static final synchronized strictfp void main(String[] args) { } // ‚úÖ with modifiers

üîÑ Changes in Java Versions:

  • Java 1.6: Static blocks execute even without main() method
  • Java 1.7+: Main method is required to start execution
  • From 1.7, better error messages if main() not found
üí° Command Line Arguments: Arguments passed from command prompt are available in args array.
Example: java MyProgram arg1 arg2 ‚Üí args[0] = "arg1", args[1] = "arg2"
