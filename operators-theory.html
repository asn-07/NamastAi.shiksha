Increment & Decrement Operators

Prefix (++x) and Postfix (x++) operators

üìã Key Points:

Prefix (++x): Increment first, then use value

Postfix (x++): Use value first, then increment

Expression Initial value of x Value of y Final value of x
y = ++x 10 11 11
y = x++ 10 10 11
y = --x 10 9 9
y = x-- 10 10 9
Rule 1: Only for Variables, Not Constants

Increment/decrement operators can only be applied to variables, not constant values.

int x = 4;
int y = ++x; // Valid: x is variable
System.out.println(y); // Output: 5

int y = ++4; // Compile Error!
// C.E: unexpected type required: variable found: value
Rule 2: No Nesting Allowed

You cannot nest increment/decrement operators.

int x = 4;
int y = ++(++x); // Compile Error!
// C.E: unexpected type required: variable found: value
Rule 3: Not for final Variables

Cannot apply increment/decrement to final variables.

final int x = 4;
x++; // Compile Error!
// C.E: cannot assign a value to final variable x
Rule 4: Applicable to All Primitive Types Except boolean
int x = 10;
x++;
System.out.println(x); // Output: 11

char ch = 'a';
ch++;
System.out.println(ch); // Output: b

double d = 10.5;
d++;
System.out.println(d); // Output: 11.5

boolean b = true;
b++; // Compile Error!
// C.E: operator ++ cannot be applied to boolean
‚ö†Ô∏è Important Difference:

b++ vs b = b + 1

For increment/decrement operators, internal type casting is performed automatically:

b++ means b = (type of b)(b + 1)

Example: byte b = 10; b++; works fine, but b = b + 1; gives compile error.

Arithmetic Operators

+ - * / % operators and type promotion rules

Rule: Result Type = max(int, type of a, type of b)

When applying arithmetic operations between two variables a and b:

Operation Result Type Example
byte + byte int byte a=10; byte b=20; a+b ‚Üí int
byte + short int byte a=10; short b=20; a+b ‚Üí int
short + short int short a=10; short b=20; a+b ‚Üí int
char + char int 'a' + 'b' = 195
char + int int 'a' + 1 = 98
int + double double 10 + 5.5 = 15.5
float + double double 10.5f + 5.5 = 16.0
System.out.println('a' + 'b'); // Output: 195
System.out.println('a' + 1); // Output: 98
System.out.println('a' + 1.2); // Output: 98.2
Infinity and NaN Handling
‚ö†Ô∏è Critical Difference:

Integral Arithmetic (byte, short, int, long):

  • No way to represent infinity ‚Üí ArithmeticException
  • No way to represent undefined results ‚Üí ArithmeticException

Floating Point Arithmetic (float, double):

  • Can represent infinity: POSITIVE_INFINITY, NEGATIVE_INFINITY
  • Can represent undefined: NaN (Not a Number)
  • No ArithmeticException for /0 or 0/0
// Integral Arithmetic - Throws Exception
System.out.println(10 / 0); // ArithmeticException: / by zero
System.out.println(0 / 0); // ArithmeticException: / by zero

// Floating Point Arithmetic - No Exception
System.out.println(10 / 0.0); // Output: Infinity
System.out.println(-10 / 0.0); // Output: -Infinity
System.out.println(0.0 / 0.0); // Output: NaN
System.out.println(-0.0 / 0.0); // Output: NaN
NaN Properties

For any value x (including NaN), comparisons with NaN always return false:

System.out.println(10 < Float.NaN); // false
System.out.println(10 <= Float.NaN); // false
System.out.println(10 > Float.NaN); // false
System.out.println(10 >= Float.NaN); // false
System.out.println(10 == Float.NaN); // false
System.out.println(Float.NaN == Float.NaN); // false

System.out.println(10 != Float.NaN); // true
System.out.println(Float.NaN != Float.NaN); // true
üìå ArithmeticException Summary:
  1. RuntimeException, not compile-time error
  2. Occurs only in integral arithmetic (byte, short, int, long)
  3. Only operations causing ArithmeticException: / and %
  4. Does NOT occur in floating-point arithmetic

String Concatenation Operator (+)

Java's only overloaded operator

üéØ Key Rule:

The + operator is the only overloaded operator in Java:

  • If at least one argument is String ‚Üí String concatenation
  • If both arguments are numeric ‚Üí Arithmetic addition
Examples of String Concatenation
String a = "ashok";
int b = 10, c = 20, d = 30;

System.out.println(a + b + c + d); // Output: ashok102030
System.out.println(b + c + d + a); // Output: 60ashok
System.out.println(b + c + a + d); // Output: 30ashok30
System.out.println(b + a + c + d); // Output: 10ashok2030
Common Errors
String a = "ashok";
int b = 10, c = 20, d = 30;

// COMPILE ERRORS:
a = b + c + d; // C.E: incompatible types
// found: int, required: java.lang.String

b = a + c + d; // C.E: incompatible types
// found: String, required: int
‚ö†Ô∏è Important:

Valid Operations:

  • a = a + b + c; ‚úì (String + int + int = String)
  • b = b + c + d; ‚úì (int + int + int = int)

Invalid Operations:

  • a = b + c + d; ‚úó (can't assign int to String)
  • b = a + c + d; ‚úó (can't assign String to int)

Relational Operators

<, <=, >, >= comparison operators

Applicable to All Primitive Types Except boolean
System.out.println(10 > 10.5); // false
System.out.println('a' > 95.5); // true
System.out.println('z' > 'a'); // true
System.out.println(true > false); // Compile Error!
// C.E: operator > cannot be applied to boolean, boolean
Not Applicable to Object Types
System.out.println("bhaskar" > "bhaskar"); // Compile Error!
// C.E: operator > cannot be applied to String, String
No Nesting Allowed
System.out.println(10 < 20 < 30); // Compile Error!
// C.E: operator < cannot be applied to boolean, int
// (10 < 20) returns boolean, then (boolean < 30) is invalid

Equality Operators

== and != operators for primitives and objects

Applicable to All Primitive Types Including boolean
System.out.println(10 == 20); // false
System.out.println('a' == 'b'); // false
System.out.println('a' == 97.0); // true
System.out.println(false == false); // true
For Objects: Reference/Address Comparison

r1 == r2 returns true only if both references point to the same object.

Thread t1 = new Thread();
Thread t2 = new Thread();
Thread t3 = t1;

System.out.println(t1 == t2); // false
System.out.println(t1 == t3); // true
Must Have Type Relationship Between Objects
Thread t = new Thread();
Object o = new Object();
String s = new String("durga");

System.out.println(t == o); // false (valid: Thread is subclass of Object)
System.out.println(o == s); // false (valid: String is subclass of Object)
System.out.println(s == t); // Compile Error!
// C.E: incompatible types: String and Thread (no relationship)
Null Comparisons
String s = new String("ashok");
System.out.println(s == null); // false

String r = null;
System.out.println(r == null); // true
System.out.println(null == null); // true
üìå == vs .equals()

== operator: Reference comparison (address comparison)

.equals() method: Content comparison (overridden in String, wrapper classes)

String s1 = new String("ashok");
String s2 = new String("ashok");

System.out.println(s1 == s2); // false (different objects)
System.out.println(s1.equals(s2)); // true (same content)

instanceof Operator

Type checking at runtime

Basic Usage

Syntax: objectReference instanceof ClassName/InterfaceName

Thread t = new Thread();
System.out.println(t instanceof Thread); // true
System.out.println(t instanceof Object); // true
System.out.println(t instanceof Runnable); // true (Thread implements Runnable)
Must Have Type Relationship

Compile-time requirement: There must be some relationship between the types (child-parent, parent-child, or same type).

String s = new String("bhaskar");
System.out.println(s instanceof Thread); // Compile Error!
// C.E: inconvertible types found: String, required: Thread
Parent Object is Child Type ‚Üí false
Object o1 = new Object();
System.out.println(o1 instanceof String); // false

Object o2 = new String("ashok");
System.out.println(o2 instanceof String); // true
null instanceof X ‚Üí Always false
System.out.println(null instanceof String); // false
System.out.println(null instanceof Object); // false
System.out.println(null instanceof Runnable); // false

Bitwise Operators

&, |, ^, ~ operators for booleans and integrals

Boolean Bitwise Operators

& (AND): true only if both operands are true

| (OR): true if at least one operand is true

^ (XOR): true if operands are different

System.out.println(true & false); // false
System.out.println(true | false); // true
System.out.println(true ^ false); // true
Integral Bitwise Operators
System.out.println(4 & 5); // 4
System.out.println(4 | 5); // 5
System.out.println(4 ^ 5); // 1

// Binary representation:
// 4 = 100, 5 = 101
// 4 & 5 = 100 & 101 = 100 = 4
// 4 | 5 = 100 | 101 = 101 = 5
// 4 ^ 5 = 100 ^ 101 = 001 = 1
Bitwise Complement (~) Operator

Applicable only for integral types, NOT for boolean.

System.out.println(~4); // -5
System.out.println(~true); // Compile Error!
// C.E: operator ~ cannot be applied to boolean
Boolean Complement (!) Operator

Applicable only for boolean types, NOT for integral types.

System.out.println(!true); // false
System.out.println(!false); // true
System.out.println(!4); // Compile Error!
// C.E: operator ! cannot be applied to int
üìã Summary:
Operator Applicable For Not Applicable For
&, |, ^ Both boolean and integral types -
~ Integral types only boolean
! boolean types only integral types

Short Circuit Operators

&& and || vs & and |

Differences Between &,| and &&,||
Aspect & , | && , ||
Argument Evaluation Both arguments evaluated always Second argument evaluation is optional
Performance Relatively low Relatively high
Applicable Types Both integral and boolean types Only boolean types
Evaluation Rules

&& (AND): y evaluated ONLY if x is true

|| (OR): y evaluated ONLY if x is false

int x = 10, y = 15;
if (++x < 10 && ++y > 15) {
    x++;
} else {
    y++;
}
System.out.println("x=" + x + " y=" + y);
// Output with &&: x=11 y=16 (y++ executed, ++y NOT evaluated)
Avoiding Exceptions with Short Circuit
int x = 10;
if (++x < 10 && (x/0) > 10) {
    System.out.println("Hello");
} else {
    System.out.println("Hi");
}
// Output: Hi (no ArithmeticException because (x/0) not evaluated)
üéØ Performance Example:
// With & and | (ALWAYS evaluate both):
if (method1() & method2()) { ... } // Both methods called

// With && and || (CONDITIONALLY evaluate second):
if (method1() && method2()) { ... } // method2 called only if method1 true

Type Cast Operators

Implicit and explicit type casting

Implicit Type Casting (Widening)
  • Compiler performs automatically
  • Lower datatype ‚Üí Higher datatype
  • No loss of information
  • Also called Upcasting
int x = 'a'; // char to int (97)
System.out.println(x); // Output: 97

double d = 10; // int to double
System.out.println(d); // Output: 10.0
Explicit Type Casting (Narrowing)
  • Programmer must specify
  • Higher datatype ‚Üí Lower datatype
  • May lose information
  • Also called Downcasting
int x = 130;
byte b = (byte)x; // Explicit cast required
System.out.println(b); // Output: -126

byte b = x; // Compile Error!
// C.E: possible loss of precision
Type Casting Rules
// Most significant bits lost when casting to smaller type
int x = 150;
short s = (short)x;
byte b = (byte)x;
System.out.println(s); // 150
System.out.println(b); // -106

// Decimal points lost when casting floating to integral
double d = 130.456;
int i = (int)d;
System.out.println(i); // 130
üìã Type Casting Hierarchy:

Implicit (no cast needed): byte ‚Üí short ‚Üí int ‚Üí long ‚Üí float ‚Üí double

Explicit (cast required): double ‚Üí float ‚Üí long ‚Üí int ‚Üí short ‚Üí byte

char: Can implicitly cast to int/long/float/double, but requires explicit cast from other types

Assignment Operators

=, +=, -=, *=, /=, %= operators

1. Simple Assignment
int x = 10; // Simple assignment
2. Chained Assignment
int a, b, c, d;
a = b = c = d = 20; // Valid chained assignment
System.out.println(a + "---" + b + "---" + c + "---" + d);
// Output: 20---20---20---20
‚ö†Ô∏è Cannot Chain at Declaration:
int a = b = c = d = 30; // Compile Error!
// C.E: cannot find symbol (variables b, c, d not declared)
3. Compound Assignment Operators

Mix assignment with other operators:

Operator Example Equivalent To
+= a += 20 a = a + 20
-= a -= 5 a = a - 5
*= a *= 3 a = a * 3
/= a /= 2 a = a / 2
%= a %= 3 a = a % 3
&= a &= 2 a = a & 2
|= a |= 2 a = a | 2
^= a ^= 2 a = a ^ 2
üéØ Internal Type Casting in Compound Assignment:

Compound assignment operators perform automatic type casting:

byte b = 10;
b += 1; // Valid: b = (byte)(b + 1)

b = b + 1; // Compile Error!
// C.E: possible loss of precision
// b+1 returns int, can't assign to byte without cast

Conditional Operator (? :)

Java's only ternary operator

Basic Syntax

condition ? expression1 : expression2

int x = (10 > 20) ? 30 : 40;
System.out.println(x); // Output: 40
Nesting Allowed
int x = (10 > 20) ? 30 : ((40 > 50) ? 60 : 70);
System.out.println(x); // Output: 70
Type Compatibility Issue
int a = 10, b = 20;
byte c1 = (a > b) ? 30 : 40; // Compile Error!
// C.E: possible loss of precision
// 30 and 40 are int literals, result is int
üìå Solution:
byte c1 = (a > b) ? (byte)30 : (byte)40; // Valid

new Operator & Related Concepts

Object creation, arrays, and runtime type checking

new Operator

Used to create objects:

String s = new String("Java");
Thread t = new Thread();

Note: There is NO delete operator in Java (garbage collector handles cleanup).

[ ] Operator (Arrays)

Used to declare/create arrays:

int[] arr = new int[5]; // Array creation
arr[0] = 10; // Array element access
Java Operator Precedence
  1. Unary operators: [], x++, x--, ++x, --x, +x, -x, ~, !, new, (type)
  2. Arithmetic: *, /, %, +, -
  3. Shift: >>, >>>, <<< /li>
  4. Comparison: <, <=,>, >=, instanceof
  5. Equality: ==, !=
  6. Bitwise: &, ^, |
  7. Short circuit: &&, ||
  8. Conditional: ? :
  9. Assignment: =, +=, -=, *=, /=, %=, etc.
Evaluation Order of Operands

Operands are evaluated from left to right before applying any operator:

class Test {
    public static void main(String[] args) {
        System.out.println(m1(1) + m1(2) * m1(3) / m1(4) + m1(5) * m1(6));
    }
    public static int m1(int i) {
        System.out.println(i);
        return i;
    }
}
// Output: 1 2 3 4 5 6 (all operands evaluated left to right first)
// Then: 1 + (2*3)/4 + (5*6) = 1 + 1 + 30 = 32

Complex Examples

Tricky operator combinations and evaluations

Example 1: x = x++ evaluation
int x = 10;
x = x++;
System.out.println(x); // Output: 10

// Evaluation steps:
// 1. Consider old value of x for assignment: x = 10
// 2. Increment x: x becomes 11
// 3. Perform assignment with old considered value: x = 10
Example 2: Compound assignment with multiple increments
int i = 1;
i += ++i + i++ + ++i + i++;
System.out.println(i); // Output: 13

// Evaluation:
// i = i + ++i + i++ + ++i + i++
// i = 1 + 2 + 2 + 4 + 4
// i = 13
