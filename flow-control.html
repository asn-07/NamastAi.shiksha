Introduction to Flow Control

Order of statement execution at runtime

üìã Key Points:

Flow control describes the order in which statements will be executed at runtime.

Java provides three types of flow control statements:

  1. Selection Statements: if-else, switch
  2. Iterative Statements: while, do-while, for, for-each
  3. Transfer Statements: break, continue, return

Selection Statements (if-else)

Conditional execution based on boolean expressions

Syntax and Basic Rules
// Basic if-else syntax
if (boolean_expression) {
    // executed if true
} else {
    // executed if false
}
‚ö†Ô∏è Important Rule:

The argument to the if statement must be boolean. Providing any other type results in compile-time error.

Example: Compile Error with Non-boolean
public class ExampleIf {
    public static void main(String[] args) {
        int x = 0;
        if(x) { // COMPILE ERROR!
            System.out.println("hello");
        } else {
            System.out.println("hi");
        }
    }
}
// Output: Compile time error: incompatible types
// found: int, required: boolean
Curly Braces and Single Statements
üìå Rule:

Curly braces are optional for single statements, but declarative statements are not allowed without braces.

// Valid - single statement without braces
if (true)
    System.out.println("Hello");

// Invalid - declarative statement without braces
if (true)
    int x = 10; // COMPILE ERROR!
// Error: 'class' expected
Semicolon as Empty Statement
if (true); // Valid empty statement
{
    System.out.println("Hello");
}
// Output: Hello (always executes, not part of if)
3

Switch Statement

Multi-way branching with fall-through behavior

Basic Syntax
switch (expression) {
    case value1:
        // statements
        break;
    case value2:
        // statements
        break;
    default:
        // statements
}
Allowed Switch Argument Types
Until Java 1.4 From Java 1.5 Onwards
byte, short, char, int byte, short, char, int
Byte, Short, Character, Integer (wrapper classes)
enum types
‚ö†Ô∏è Not Allowed:

NOT allowed: boolean, long, float, double, String (until Java 1.6)

String allowed from Java 1.7+

Case Label Rules
üìå Case Label Requirements:
  1. Must be compile-time constant
  2. Can be expressions but must be constant expressions
  3. Value must be within range of switch argument type
  4. Duplicate case labels not allowed
int x = 10;
final int y = 20; // final makes it constant
switch (x) {
    case 10: // Valid - constant
    case y: // Valid - final variable
    case 10+20: // Valid - constant expression
}
Fall-Through Behavior
üéØ Fall-Through Concept:

Within switch statement, if any case is matched, from that case onwards all statements will be executed until end of switch or break statement.

Advantage: Can define common action for multiple cases.

int x = 0;
switch (x) {
    case 0:
        System.out.println("0");
    case 1:
        System.out.println("1");
        break;
    case 2:
        System.out.println("2");
    default:
        System.out.println("default");
}
// Output when x=0: 0, 1 (falls through to case 1, breaks)
// Output when x=2: 2, default (falls through to default)
Default Case
üìå Default Case Rules:
  • Can be taken only once within a switch
  • Executed only if no other case matches
  • Can be placed anywhere, but convention is last
  • Still participates in fall-through

While Loop

Pre-test loop - condition checked before execution

Syntax and Basic Usage
// Basic while loop syntax
while (boolean_expression) {
    // loop body
}

// Best for unknown iterations in advance
while (rs.next()) { ... }
while (e.hasMoreElements()) { ... }
while (itr.hasNext()) { ... }
Boolean Requirement
while (1) { // COMPILE ERROR!
    System.out.println("hello");
}
// Error: incompatible types: int cannot be converted to boolean
Unreachable Statements in While
‚ö†Ô∏è Unreachable Code:

Compiler detects unreachable statements in loops:

  • while(true) { ... } - statements after are unreachable
  • while(false) { ... } - loop body is unreachable
while (true) {
    System.out.println("hello");
}
System.out.println("hi"); // COMPILE ERROR!
// Error: unreachable statement
Final Variables and Compiler Optimization
üìå Compiler Behavior with Final Variables:
  • Final variables replaced with values at compile time
  • Operations with only constants performed by compiler
  • Operations with variables performed at runtime by JVM
final int a = 10, b = 20;
while (a < b) {
    System.out.println("hello");
}
System.out.println("hi"); // COMPILE ERROR!
// Compiler knows a
// So statements after loop are unreachable
5

Do-While Loop

Post-test loop - executes at least once

Syntax and Basic Usage
// Basic do-while syntax
do {
    // loop body
} while (boolean_expression);

// Executes at least once
int x = 10;
do {
    System.out.println("Hello");
    x++;
} while (x < 15);
Unreachable Statements in Do-While
do {
    System.out.println("hello");
} while (true);
System.out.println("hi"); // COMPILE ERROR!
// Error: unreachable statement
Do-While vs Continue (Dangerous Combination)
‚ö†Ô∏è Most Dangerous Combination:

Using continue in do-while can create infinite loops or unexpected behavior.

int x = 0;
do {
    x++;
    System.out.println(x);
    if (++x < 5)
        continue;
    System.out.println(x);
} while (++x < 10);
// Output: 1, 4, 6, 8, 10

For Loop

Most commonly used loop when iterations are known

Syntax and Structure
// Basic for loop syntax
for (initialization; condition; increment/decrement) {
    // loop body
}

// All three parts are independent and optional
for (;;) {
    System.out.println("infinite loop");
}
Initialization Section
üìå Initialization Rules:
  • Executed only once
  • Can declare multiple variables of same type
  • Can take any valid Java statement including System.out.println()
int i = 0;
for (System.out.println("hello u r sleeping"); i < 3; i++) {
    System.out.println("no boss, u only sleeping");
}
// Output: hello u r sleeping (once)
// no boss, u only sleeping (3 times)
Condition and Increment Sections
// Conditional expression optional (defaults to true)
for (int i = 0; ; i++) {
    // infinite loop
}

// Can use any statement in increment section
int i = 0;
for (System.out.println("hello"); i < 3; System.out.println("hi")) {
    i++;
}
// Output: hello, hi, hi, hi
Unreachable Statements in For Loop
for (int i = 0; true; i++) {
    System.out.println("hello");
}
System.out.println("hi"); // COMPILE ERROR!
// Error: unreachable statement
7

Enhanced For Loop (For-Each)

Introduced in Java 1.5 for arrays and collections

Syntax and Basic Usage
// For-each syntax
for (type variable : collection_or_array) {
    // process variable
}

// Example with array
int[] arr = {10, 20, 30, 40, 50};
for (int value : arr) {
    System.out.println(value);
}
// Output: 10 20 30 40 50
Iterable Requirement
üìå Target Requirement:

The target element in for-each loop should be an Iterable object.

An object is iterable if its class implements java.lang.Iterable interface.

// Iterable interface (Java 1.5+)
public interface Iterable {
    Iterator iterator();
}

// Every array and Collection implements Iterable
Limitations of For-Each
‚ö†Ô∏è Limitations:
  • Not a general-purpose loop
  • Can only iterate forward (left to right)
  • Cannot modify collection during iteration (ConcurrentModificationException)
  • No access to index
// Normal for loop - can go both directions
for (int i = 0; i < arr.length; i++) { ... } // forward
for (int i = arr.length-1; i >= 0; i--) { ... } // backward

// For-each - only forward
for (int value : arr) { ... } // only left to right

Break Statement

Transfer control out of loops and switch

Three Uses of Break
Use Case Example Purpose
Inside switch case 1: ... break; Stop fall-through
Inside loops while(true) { if(cond) break; } Exit loop early
Inside labeled blocks l1: { ... break l1; } Exit labeled block
Examples of Break Usage
// 1. Inside switch to stop fall-through
int x = 0;
switch (x) {
    case 0:
        System.out.println("Hello");
        break; // stops fall-through
    case 1:
        System.out.println("Hi");
}

// 2. Inside loop to exit based on condition
for (int i = 0; i < 10; i++) {
    if (i == 5) break;
    System.out.println(i);
}
// Output: 0 1 2 3 4 (stops at 5)
Break with Labels
l1: {
    System.out.println("begin");
    int x = 10;
    if (x == 10)
        break l1;
    System.out.println("end"); // never executed
}
System.out.println("hello");
// Output: begin, hello
Break Restrictions
int x = 10;
if (x == 10)
    break; // COMPILE ERROR!
System.out.println("hello");
// Error: break outside switch or loop

Continue Statement

Skip current iteration and continue with next

Basic Usage
// Print odd numbers from 0 to 9
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) // if even
        continue; // skip even numbers
    System.out.println(i);
}
// Output: 1 3 5 7 9
Continue Restrictions
‚ö†Ô∏è Important Rule:

Continue can only be used inside loops. Using it anywhere else causes compile error.

int x = 10;
if (x == 10)
    continue; // COMPILE ERROR!
System.out.println("hello");
// Error: continue outside of loop
10

Labeled Break and Continue

Control nested loops with labels

Labeled Break Example
l1: // Label for outer loop
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i == j)
            break l1; // breaks outer loop
        System.out.println(i + "......." + j);
    }
}
// When i=0, j=0: break l1 executes, no output
Labeled Continue Example
l1: // Label for outer loop
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i == j)
            continue l1; // continues outer loop
        System.out.println(i + "......." + j);
    }
}
// Output: 0.......1, 0.......2, 1.......0, 1.......2, 2.......0, 2.......1
// Skips when i == j (diagonal elements)

Unreachable Code Detection

Compiler checks for dead code in loops

Compiler Behavior
üìå Important Note:

Compiler won't check unreachability for if-else statements (except for constant conditions). It only checks unreachability in loops.

// Unreachable in loop - Compile Error
while (true) {
    System.out.println("hello");
}
System.out.println("hi"); // COMPILE ERROR!

// Unreachable in if-else - No Error
if (true) {
    System.out.println("hello");
} else {
    System.out.println("hi"); // Unreachable but no error
}
Final Variables and Unreachable Code
final int a = 10, b = 20;
while (a < b) {
    System.out.println("hello");
}
System.out.println("hi"); // COMPILE ERROR!
// Compiler replaces final variables with values
// Knows 10 < 20 is always true
// So statement after loop is unreachable
